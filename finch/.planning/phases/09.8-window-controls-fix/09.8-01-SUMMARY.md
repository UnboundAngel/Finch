## Key Implementation Details

### 1. Tauri v2 API Alignment
- **Import Update**: Changed `import { getCurrentWindow } from '@tauri-apps/api/window'` to `import { getCurrentWindow, currentMonitor } from '@tauri-apps/api/window'`.
- **Monitor Access**: In the `snapTo` function, replaced `appWindow.currentMonitor()` with `currentMonitor()`.
- **Work Area Destructuring**: Corrected the extraction of monitor dimensions. In Tauri v2, `workArea` properties are nested under `position` (`x`, `y`) and `size` (`width`, `height`).

### 2. Tooltip Trigger Fix
- **Render Prop Implementation**: Replaced `asChild` on `TooltipTrigger` for Minimize and Close buttons with the `render` prop. 
- **Rationale**: The `asChild` pattern with a `<button>` child was causing hydration and nested element warnings because the underlying `TooltipTrigger` already provides a button-like interaction. Using the `render` prop allows us to explicitly spread the trigger props onto our custom button while maintaining our custom styles and logic.
