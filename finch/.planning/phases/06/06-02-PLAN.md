# Phase 06-02: Message Generation Stats

Implementation of a structured metrics system that captures token counts and stop reasons from Rust and displays them in a polished, interactive UI using the existing metadata infrastructure.

## Tasks

<task>
<name>Rust Stats Sentinel Implementation</name>
<files>
  - "src-tauri/src/lib.rs"
  - "src-tauri/src/anthropic.rs"
</files>
<action>
- Update `stream_message` in `lib.rs` and `stream_anthropic` in `anthropic.rs` to track `token_count`.
- Determine `stop_reason`:
  - "abort" if `abort_flag` is triggered.
  - "max_tokens" if the response ends due to length (detected via provider JSON).
  - "end_turn" as the default.
- Immediately before returning `Ok(())`, send a final string over the channel: `__STATS__:{"stop_reason":"...", "total_tokens":...}`.
</action>
<verify>
- Observe console output or temporarily log sent tokens to confirm the sentinel is the last item dispatched.
</verify>
<done>Rust backend sends structured performance metadata.</done>
</task>

<task>
<name>Frontend Hook Real-Time Processing</name>
<files>
  - "src/hooks/useAIStreaming.ts"
</files>
<action>
- Add `stats` state to `useAIStreaming` (totalTokens, tokensPerSecond, stopReason).
- Use `useRef` for internal `tokenCount` accumulator to prevent excessive re-renders.
- Record `startTime` using `performance.now()` when `streamMessage` initiates.
- In `channel.onmessage`:
  - Check if token starts with `__STATS__:`.
  - If yes: parse JSON, finalize `total_tokens` and `stop_reason`, update state.
  - If no: 
    - Increment `tokenCount.current`.
    - Throttled or completion-only: Update `tokensPerSecond` and `totalTokens` state only when the stream finishes or via the sentinel to maintain performance.
    - Pass token to `onToken` callback as normal.
- Reset stats ref and state at the beginning of each stream.
</action>
<verify>
- Confirm `onToken` is NOT called for the sentinel token.
- Verify stats state updates correctly at the end of a stream.
</verify>
<done>useAIStreaming hook captures and computes generation metrics efficiently with useRef accumulator.</done>
</task>

<task>

<files>

- Update the `onComplete` callback in `handleSend`:
  - Capture the final stats from the hook.
  - Update the latest message's `metadata` with these frozen values (tokensPerSecond, totalTokens, stopReason).
</action>
<verify>

</verify>
<done>Generation stats are persisted within individual message metadata upon completion and isLatest prop is passed.</done>
</task>
