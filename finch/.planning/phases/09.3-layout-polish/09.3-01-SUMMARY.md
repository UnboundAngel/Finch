# Phase 09.3 Plan 01: Layout Polish Summary

## One-liner
Restructured the Dashboard into a three-tier vertical design (Top Bar, Second Bar, Zone) with polished sidebars and correct drag regions.

## Key Changes
- **Three-Tier Architecture:**
  - **Top Bar (h-14):** Fixed header with window controls, theme switch, notifications, and left sidebar toggle. It is the only area that handles window dragging.
  - **Second Bar (h-12):** Middle section with ModelSelector, Unload button, and Right sidebar toggle.
  - **Zone (flex-1):** Main content area containing the Left Sidebar, Chat Area, and Right Sidebar, all constrained within the remaining viewport height.
- **Sidebar Polish:**
  - Updated `RightSidebar` to match the `bg-sidebar` color and ensured its animation respects the new layout constraints.
  - Modified `ChatSidebar` to accept a `className` prop, allowing it to be positioned absolutely within the Zone.
  - Toggled Left Sidebar now uses SVG assets for the button and controls a top-level `isLeftSidebarOpen` state.
- **Interaction Refinements:**
  - Centered Model Selector and inline Unload button for better visual balance.
  - Removed `data-tauri-drag-region` from all elements except the Top Bar, preventing accidental window dragging while interacting with UI controls.
