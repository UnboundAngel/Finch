<task type="auto">
  <name>Task 1: Create useInactivityEject hook</name>
  <files>src/hooks/useInactivityEject.ts</files>
  <action>
    Create a new hook `useInactivityEject` that manages a setTimeout based on a delay.
    - Props: `{ provider: string, modelId: string, onEject: () => void, delay?: number }`.
    - Default delay: 600,000ms (10 mins).
    - Use `useRef` for the timer ID to persist across renders.
    - Use `useCallback` for `resetTimer` to allow parent components to trigger resets without re-renders.
    - `useEffect` to start the timer on mount and cleanup on unmount/dependency change.
    - Logic: When timer fires:
      1. Check if `provider.startsWith('local_')`.
      2. If true, `invoke('eject_model', { provider, model_id: modelId }).catch(() => {})`.
      3. On success (or even if it fails silently), call `onEject()`.
    - **CRITICAL:** The `invoke` call MUST explicitly catch and swallow errors using `.catch(() => {})` to ensure failures are silent and do not disrupt the user.
    - Reference: `STATE.md` hotfix confirms `model_id` is the correct key for the Rust backend.
  </action>
  <verify>
    <automated>npx tsc --noEmit</automated>
  </verify>
  <done>Hook created with correct logic, silent error handling, and TypeScript types pass.</done>
</task>
